실습: Case Clinic — 위축 장면 분석

실제 팀 경험을 구조적으로 분해하기

🛠️ WORKSHOP

📝 트리거 분석 워크시트

① 상황 (When/Where)

예: "월요일 전체 회의에서…"

② 행동 (Who/What)

예: "담당자가 실적 미달 보고를 위해…"

③ 반응 (Response) ⚡핵심

예: "팀장이 한숨을 쉬며 '그래서?'라고 말함"

④ 결과 (Outcome)

예: "이후 아무도 미달 수치를 먼저 말하지 않음"

⏱️ 진행 타임라인

① 사건 서술 2분
② 상황 질문 3분
③ 구조 분석 5분
④ 규범 초안 5분
⑤ 합의 선언 2분
총 시간 17분

🚫 금지 행동

❌ 개인 비난 ("누구 때문에…")

❌ 해석 끼워넣기 ("그건 아마…")

❌ 시간 초과 방치

❌ 외부 유출

규범 초안 → 팀 운영 프로토콜

Case Clinic 산출물을 실행 가능한 규칙으로 전환하기

🔄 변환 공식: 사건 → 구조 → 규범 → 프로토콜

사건

"팀장 한숨 → 질문 소멸"

→

구조 진단

감정 표출 → 관찰 학습

→

규범 초안

"질문 첫 반응 = 감사"

→

프로토콜

회의록 체크리스트에 반영

✅ 좋은 규범의 5가지 조건

①

구체적

"질문에 감사합니다로 시작"

②

관찰 가능

다른 사람이 확인 가능

③

합의 기반

팀 전원 합의 필수

④

수정 가능

월례 리뷰로 업데이트

⑤

위반시 절차

제재가 아닌 피드백

📋 규범 예시 (참고용)

회의 규범

"회의 시작 5분은 전원 발언 라운드. 리더는 마지막에 말한다. 반대 의견에는 '감사합니다'로 시작."

단톡방 규범

"업무 외 시간은 '확인' 이모지만. 긴급은 전화. 읽씹 = 정상."

피드백 규범

"비판은 '행동+영향+요청' 형식. 사람이 아니라 행동에 초점."

실수 보고 규범

"실수 보고 24시간 이내. 보고자에게 감사 표현. 후속: 원인 분석만."

회식 규범

"참석 선택권. 1차 22시 종료. 불참 사유 불문. 귀가 비난 금지."

리더 안전 행동 — 말이 아니라 행동이 신호다

Leader Safety Behavior: 5가지 마이크로-행동이 팀의 발언을 바꾼다

🤫

마지막에 말하기

리더가 먼저 의견 내면 동조 발생. → "다들 먼저 들을게요"

효과: 발언 다양성 ↑

🙏

반대에 감사하기

"좋은 지적이에요. 더 얘기해 주세요" → 안전 신호 발송

효과: 이견 표현 빈도 ↑

💡

실수 먼저 공개

"지난번 내 판단이 틀렸어요" → 실수 = 안전한 것

효과: 실수 은폐 ↓

❓

질문으로 시작하기

"이 결정을 틀리게 만들 수 있는 건 뭐죠?" → 방어 해제

효과: 비판적 사고 ↑

📢

후속 조치 공유

"지난번 건의 덕에 이렇게 바뀌었습니다" → 말하면 바뀌는 증거

효과: 체념적 침묵 ↓

"리더의 마이크로-행동 5가지가…"

팀의 심리적 안전 수준을
결정한다

— Edmondson (2019)

개인 액션 플랜 — "다음 주 월요일부터"

오늘 배운 것을 하나의 행동으로 전환

📋 ACTION PLAN

📝 My Action Plan

1. 내가 발견한 우리 팀의 지뢰밭 (Top 1)

예: "주간 보고 시 실적 미달 보고"

2. 위축의 주 동기 (침묵 4유형 중)

☐ 체념 ☐ 방어 ☐ 친사회 ☐ 전략

3. 다음 주 월요일 실천할 행동 1가지

예: "회의 시작에 '다들 먼저 들을게요' 말하기"

4. 성공 기준 (2주 후 확인)

예: "회의 중 질문이 2건 이상 나옴"

💡 좋은 액션의 조건

✓ 구체적: 행동 수준까지 명시

✓ 측정 가능: 성공 기준 포함

✓ 소규모: 1가지만 집중

✓ 즉시 실행: 다음 주부터

📊 후속 추적 방법

Week 1

행동 실천 + 자기 관찰

Week 2

성공 기준 확인 + 조정

Week 3-4

팀 내 공유 + 피어 피드백
